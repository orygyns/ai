Architecture — Request Flow
1 Semantic Cache Embed query → vector similarity search. Score ≥ 0.92 → return cached answer. No Groq call. $0.00 · YOU HOST
2 LLMLingua-2 Compress prompt 4–10× before it reaches Groq. Runs on CPU. Open source (Microsoft). $0.00 · YOU HOST
3 History Compactor After 5 turns, summarize history → 50-token summary. Replaces ~400 tokens of raw context. ~$0.000002 · YOU HOST
4 Simple Router If compressed tokens < 60 → 8B. If ≥ 60 or code/math keywords → 70B. Nothing else. $0.00 · YOU HOST
5 Groq: 8B (80%) llama-3.1-8b-instant. Simple Q&A, summaries, short tasks. 500–1000 TPS. $0.05/M · USER'S KEY
6 Groq: 70B (20%) llama-3.3-70b-versatile. Complex reasoning, code, long-form. GPT-4 quality. $0.59/$0.79/M · USER'S KEY
7 Cache Write Async: store (vector, response) in pgvector. Every future similar query is free. Async · $0.00

from sentence_transformers import SentenceTransformer
model = SentenceTransformer("all-MiniLM-L6-v2")   # 22MB, CPU only

def cache_lookup(query, threshold=0.92):
    vec = model.encode(query).tolist()
    # pgvector cosine similarity query
    row = db.fetchone("SELECT response FROM cache ORDER BY embedding <=> %s LIMIT 1", [vec])
    if row and cosine_sim(vec, row.embedding) >= threshold:
        return row.response   # cache hit → $0.00
    return None

3 — History Compactor

def compact_history(history, groq_client):
    if len(history) <= 5: return history
    old = history[:-2]   # everything except last 2 turns
    summary = groq_client.chat.completions.create(
        model="llama-3.1-8b-instant",
        messages=[{"role":"user", "content": f"Summarize in 2 sentences: {old}"}],
        max_tokens=60
    ).choices[0].message.content
    return [{"role":"system", "content": f"Prior context: {summary}"}] + history[-2:]

4 — Router + Capped Groq Call

COMPLEX = ["code", "debug", "function", "math", "solve", "analyze", "compare"]

def pick_model(query):
    if len(query.split()) > 60 or any(k in query.lower() for k in COMPLEX):
        return "llama-3.3-70b-versatile"   # ~20% of queries
    return "llama-3.1-8b-instant"           # ~80% of queries

response = groq_client.chat.completions.create(

    max_tokens=200,
    stop=["###", "\n\n\n"],
    temperature=0.3

Security — Non-Negotiable
  • AES-256-GCM encrypt every Groq key in Supabase. Never log plaintext keys.
  • Frontend only receives your platform session token — never the Groq key.
  • Rate limit: 60 requests/min per user via Upstash Redis sliding window.
  • Enable Supabase Row Level Security — users can only read their own data.
  • HTTPS enforced everywhere (Vercel + Render handle TLS automatically).
  • Prompt users to rotate Groq key every 90 days — show a dashboard banner.
Strategy — PWA First, Native Later
Approach Progressive Web App (PWA). Same codebase as your website. Users install from browser — no App Store needed.
Cost $0. No Apple Developer account ($99/yr) or Google Play ($25 one-time) until you choose to go native.
Install Android Chrome: automatic "Add to Home Screen" prompt. iOS Safari: Share → Add to Home Screen.
When to go native 500+ users requesting it, or revenue > $100/mo. Wrap with Capacitor.js — one command turns your Next.js app into an iOS/Android bundle.
